## 七、技术栈决策

| 维度 | 决策 | 理由 |
|---|---|---|
| 框架 | **继续纯静态 HTML/CSS** | deploy 脚本是 wrangler pages，没构建流程；引入框架会破坏 monorepo 同步 |
| CSS | **手写 CSS + CSS 变量**（**不**用 Tailwind CDN）| cleartok 用了 Tailwind，但我们页面有限，手写更可控、更轻 |
| JS | **零 JS**（首版），时间估算器用纯 HTML+CSS | 静态站无需交互框架 |
| 字体 | **Inter**（CDN 加载）| cleartok 同款 |
| 图标 | **Lucide Icons**（CDN 内嵌 SVG）| cleartok 用了 Lucide React，我们用纯 SVG |
| 多语言 | 首版只做**英文**，加 `<link rel="alternate" hreflang="x">` 占位 | 后续按需扩展 |
| 部署 | 保持 `wrangler pages deploy` 流程 | — |
| 同步 | 不进 `sync-shared.js`（它是产物，不是源）| — |
